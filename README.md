# 常熟智游 Web 端原型

这是一个零依赖静态 Web App 原型，围绕“常熟全域文旅智能出行助手”的五个核心能力搭建：

- AI 行程规划
- 实时文旅地图
- 停车与客流预测
- 景区一码通与智能讲解
- 诉求闭环处理

工程内不包含外部 clone 项目，页面、数据层、交互脚本和视觉资产均为本工程自有文件。旅行类开源项目只作为设计方向参考，没有保留在仓库目录中。

## 打开方式

直接双击 `index.html` 即可在浏览器打开。

也可以启动本地静态服务：

```bash
python -m http.server 5173
```

然后访问：

```text
http://127.0.0.1:5173/
```

## 文件说明

- `index.html`：页面结构与五个核心页面段落
- `styles.css`：视觉设计、响应式布局、滚动动效、地图/手机样机样式
- `data/content.js`：页面数据源，更新行程、地图点位、停车余位、讲解、工单示例等内容
- `app.js`：读取 `data/content.js` 并渲染页面，处理 AI 行程模拟、地图筛选、停车联动、讲解切换、工单提交交互
- `assets/`：本地生成的常熟湖山/古城/芦苇荡风格视觉资产

## 当前交互

- 首页目的地卡片：点击虞山、尚湖、沙家浜卡片，会把对应需求写入 AI 输入并刷新行程
- 行程模式：经典一日、银发慢游、亲子周末、红色研学可切换
- 路线指标：根据路线类型更新总时长、步行距离、停车/休息/研学指标
- 实时地图：可筛选点位，也可开关客流热力、停车余位、便民服务图层
- 停车客流：选择停车场或预测时段，会刷新停车建议
- 景区讲解：可切换虞山、尚湖、沙家浜讲解，并模拟播放/暂停
- 诉求闭环：提交工单后生成编号，可推进或重置处理进度

## 如何更新内容

当前版本已经从“写死页面”改成“数据驱动页面”。日常演示更新只需要改：

```text
data/content.js
```

常见更新位置：

- `todayStatus`：首页实时状态
- `todayAdvice`：首页避峰建议
- `services`：服务入口宫格
- `routes`：AI 行程路线模板
- `mapPins`：地图点位、客流指数、停车余位
- `parkingLots`：停车场列表
- `guides`：景区讲解内容
- `ticket`：诉求工单示例
- `management`：管理驾驶舱数据

后续如果要接真实后台，可以让后端输出同样结构的 JSON，然后把 `data/content.js` 替换为接口请求，例如 `/api/travel-dashboard`。

## 已验证
